Typo tolerance is a Meilisearch feature that returns relevant results even when search queries contain spelling mistakes. Instead of showing "no results found," Meilisearch understands what users meant and finds matching documents.

## How typo tolerance works

Meilisearch calculates the number of typos between query terms and indexed words using edit distance (Damerau-Levenshtein). The fewer changes needed to transform one word into another, the closer the match.

By default, Meilisearch applies these rules:

| Query word length | Typos allowed |
|-------------------|---------------|
| 1-4 characters | 0 typos |
| 5-8 characters | 1 typo |
| 9+ characters | 2 typos |

## Examples

| Query | Matches | Why |
|-------|---------|-----|
| `lost` → `last` | No match | 4 characters, no typos allowed |
| `greeec` → `greece` | Match | 6 characters, 1 typo allowed |
| `philisophie` → `philosophy` | Match | 11 characters, 2 typos allowed |

## Configuring typo tolerance

You can customize typo tolerance through the [settings API](/reference/api/settings/get-typotolerance):

```json
PATCH /indexes/books/settings/typo-tolerance
{
  "enabled": true,
  "minWordSizeForTypos": {
    "oneTypo": 5,
    "twoTypos": 9
  },
  "disableOnWords": ["ISBN", "SKU"],
  "disableOnAttributes": ["product_id", "email"]
}
```

### Configuration options

| Setting | Description |
|---------|-------------|
| `enabled` | Enable or disable typo tolerance entirely |
| `minWordSizeForTypos` | Customize word length thresholds |
| `disableOnWords` | Exact terms that should never allow typos |
| `disableOnAttributes` | Fields where typo tolerance is disabled |

## When to disable typo tolerance

Disable typo tolerance for:

- **Identifiers**: Product codes, ISBNs, email addresses
- **Technical terms**: Where precision matters more than fuzzy matching
- **Short codes**: SKUs, postal codes, abbreviations

## Typo ranking rule

Typo tolerance works together with the `typo` [ranking rule](/getting_started/glossary/ranking_rules). Documents with fewer typos rank higher than those with more typos. You can reorder or remove this ranking rule to change how typos affect result ordering.
