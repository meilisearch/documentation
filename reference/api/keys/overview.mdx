The `/keys` route allows you to create, manage, and delete API keys.

<Warning>
To use these endpoints, you must first [set a master key](/products/platform/basic_security). All key management operations require the master key for authorization.
</Warning>

## Key object

```json
{
  "name": "Default Search API Key",
  "description": "Use it to search from the frontend code",
  "key": "0a6e572506c52ab0bd6195921575d23092b7f0c284ab4ac86d12346c33057f99",
  "uid": "74c9c733-3368-4738-bbe5-1d18a5fecb37",
  "actions": ["search"],
  "indexes": ["*"],
  "expiresAt": null,
  "createdAt": "2021-08-11T10:00:00Z",
  "updatedAt": "2021-08-11T10:00:00Z"
}
```

## Default keys

When you set a master key, Meilisearch automatically creates two default API keys:

| Key | Actions | Use case |
|:----|:--------|:---------|
| **Default Search API Key** | `search` | Frontend search queries |
| **Default Admin API Key** | `*` | Backend operations |
