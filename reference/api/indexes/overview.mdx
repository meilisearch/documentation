The `/indexes` route allows you to create, manage, and delete your indexes.

[Learn more about indexes.](/getting_started/glossary/indexes)

## Index object

```json
{
  "uid": "movies",
  "createdAt": "2022-02-10T07:45:15.628261Z",
  "updatedAt": "2022-02-21T15:28:43.496574Z",
  "primaryKey": "id"
}
```

| Name | Type | Description |
|:-----|:-----|:------------|
| **`uid`** | String | Unique identifier of the index. Once created, it cannot be changed |
| **`createdAt`** | String | Creation date in RFC 3339 format |
| **`updatedAt`** | String | Latest update date in RFC 3339 format |
| **`primaryKey`** | String/null | Primary key of the index |

<Note>
An index is automatically created when you add documents if it doesn't exist yet. You only need to explicitly create an index if you want to set the primary key beforehand.
</Note>
