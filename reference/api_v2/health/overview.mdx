The `/health` route allows you to verify that Meilisearch is running and ready to accept requests.

<Tip>
Use this endpoint for health checks in container orchestration systems like Kubernetes, Docker Swarm, or load balancers.
</Tip>

## Response

A healthy Meilisearch instance returns:

```json
{
  "status": "available"
}
```

## Use cases

- **Kubernetes**: Liveness and readiness probes
- **Load Balancers**: Backend health verification
- **Monitoring**: Uptime checks and alerting
