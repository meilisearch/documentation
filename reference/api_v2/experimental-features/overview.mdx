The `/experimental-features` route allows you to manage experimental features in Meilisearch.

<Warning>
Experimental features may change or be removed without notice. Use in production at your own risk.
</Warning>

## Available experimental features

| Feature | Description |
|:--------|:------------|
| **Logs route** | Stream logs via API |
| **Metrics** | Prometheus-compatible metrics endpoint |
| **Network** | Multi-node deployment configuration |
| **Vector store** | Advanced vector storage settings |
| **Edit documents by function** | Modify documents with Rhai scripts |

## Enabling features

Features can be enabled via:
1. **API**: Using the PATCH endpoint below
2. **CLI**: Using `--experimental-*` flags
3. **Environment**: Using `MEILI_EXPERIMENTAL_*` variables
