The `/network` route allows you to configure network topology for distributed Meilisearch deployments.

<Warning>
This is an experimental feature for multi-node deployments.
</Warning>

## Use cases

- **Horizontal Scaling**: Distribute load across multiple nodes
- **High Availability**: Configure failover and redundancy
- **Sharding**: Partition indexes across nodes
