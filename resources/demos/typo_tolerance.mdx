This demo lets you experiment with Meilisearch's typo tolerance settings. See how different configurations affect search results when users make spelling mistakes.

## Features demonstrated

- **Configurable tolerance**: Adjust typo limits per word length
- **Side-by-side comparison**: See results with different settings
- **Real-time updates**: Changes apply instantly
- **Edge cases**: Test with intentional misspellings

## Use case

Typo tolerance is essential for good search UX:

- Users type quickly and make mistakes
- Mobile keyboards lead to more typos
- International users may not know exact spellings
- Product names and technical terms are often misspelled

## How typo tolerance works

Meilisearch calculates the Levenshtein distance (number of edits) between the query and indexed words:

| Word length | Default typos allowed |
|-------------|----------------------|
| 1-4 characters | 0 typos |
| 5-8 characters | 1 typo |
| 9+ characters | 2 typos |

## Configuration options

```json
{
  "typoTolerance": {
    "enabled": true,
    "minWordSizeForTypos": {
      "oneTypo": 5,
      "twoTypos": 9
    },
    "disableOnWords": ["exact_match_required"],
    "disableOnAttributes": ["sku", "id"]
  }
}
```
