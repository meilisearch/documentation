Meilisearch Cloud provides built-in monitoring dashboards so you can understand how your search is performing and catch issues before they affect users. Monitoring is organized into three areas:

| Area | What it covers |
|------|---------------|
| **Search performance** | Latency (p50/p95/p99), queries per second, error rate |
| **Indexing performance** | Indexing speed, task queue depth, per-task breakdown |
| **Usage metrics** | Memory, storage, and CPU utilization |

## Why monitoring matters

Search performance often degrades gradually rather than failing suddenly. Monitoring helps you:

- Detect latency regressions after a settings change or data update
- Identify indexing bottlenecks before the task queue backs up
- Know when your project is approaching its resource limits and needs a plan upgrade
