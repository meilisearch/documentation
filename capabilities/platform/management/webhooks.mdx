Webhooks let Meilisearch Cloud notify an external HTTP endpoint whenever a task reaches a terminal state (succeeded or failed). Use webhooks to trigger downstream actions automatically, such as purging a cache after a successful index update or alerting your team when a task fails.

## How webhooks work

When a task completes, Meilisearch Cloud sends an HTTP POST request to your configured URL. The request body is [ndjson](https://ndjson.org/) (newline-delimited JSON), with one task object per line. This matches the task object format returned by the [Tasks API](/reference/api/tasks/get-task).

```
{"uid":1,"indexUid":"movies","status":"succeeded","type":"documentAdditionOrUpdate",...}
{"uid":2,"indexUid":"movies","status":"failed","type":"documentAdditionOrUpdate",...}
```

Your endpoint must respond with a 2xx HTTP status code. Non-2xx responses are treated as delivery failures. Meilisearch Cloud retries failed deliveries with exponential backoff.

## Configuring webhooks via the Cloud UI

1. Open your project in the Meilisearch Cloud dashboard.
2. Go to **Settings** and find the **Webhooks** section.
3. Click **Add webhook**.
4. Enter the **URL** of your endpoint.
5. Optionally, enter a value for the **Authorization** header. Meilisearch Cloud will send this header with every webhook request so your endpoint can verify the source.
6. Click **Save**.

You can configure multiple webhooks for the same project. Each webhook receives all task completion events for the project.

## Configuring webhooks via the API

You can also manage webhooks programmatically using the `/webhooks` API route. See the [API reference](/reference/api/webhooks/get-webhooks) for endpoint details.

## Securing your webhook endpoint

Always validate the `Authorization` header on incoming webhook requests. Set a long, random secret value and verify it matches before processing the payload. Reject requests that do not match.

Example (Node.js):

```js
app.post('/webhook', (req, res) => {
  const auth = req.headers['authorization'];
  if (auth !== process.env.WEBHOOK_SECRET) {
    return res.status(401).end();
  }
  // Process the ndjson payload
  res.status(200).end();
});
```

## Example use cases

| Use case | Description |
|----------|-------------|
| **Cache purge** | After a `documentAdditionOrUpdate` task succeeds, invalidate your CDN or application cache |
| **Slack notification** | Send a message to a Slack channel when any task fails |
| **CI/CD integration** | Signal a deployment pipeline that a new index is ready |
| **Audit log** | Append every task event to an external log store for compliance |
