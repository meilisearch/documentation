# v0.22.0

Released: 2021-09-13

## Breaking changes

- New syntax for custom ranking rules:
```js
# new syntax
"price:asc"

# old syntax (deprecated)
"asc(price)"
```

- New indexer integration
We completely rewrote the indexer in order to:
  -  Fix crashes and RAM consumption issues
  -  Increase indexation speed
  -  Fix indexation bugs

Using the new indexer requires re-indexation of all your documents. This will be done automatically when migrating your data from a previous version. For more information on how to update to the latest version of MeiliSearch, [read our dedicated guide](https://docs.meilisearch.com/create/how_to/updating.html#update-to-the-latest-meilisearch-version).

## New features

- Sort at query time. For more information, read [our dedicated guide](https://docs.meilisearch.com/reference/features/sorting.html)
⚠️ If you import your data via a dump, the `sort` ranking rule must be added manually. Check your ranking rules via the [`GET indexes/:ui/settings/ranking-rules` route](https://docs.meilisearch.com/reference/api/ranking_rules.html#get-ranking-rules). If the `sort` is not listed, you can add it to the 5th position using the [`POST indexes/:uid/settings` route](https://docs.meilisearch.com/reference/api/ranking_rules.html#update-ranking-rules).
- Make the [download script](https://install.meilisearch.com) compatible with Windows  (requires Cygwin or equivalent)
- Custom ranking rules now work with fields containing strings, not only numbers
