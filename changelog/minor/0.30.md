# v0.30

First released: 2022-11-28

## New Features

### Improved Search Result Navigation
You can now paginate search results more effectively with the new parameters `page` and `hitsPerPage`. This allows you to retrieve an exact count of `totalHits` and `totalPages`, making it easier to create user-friendly pagination interfaces.

Example request to fetch the second page of results:

```bash
curl \
 -X POST 'http://localhost:7700/indexes/movies/search' \
 -H 'Content-Type: application/json' \
 --data-binary '{ "q": "shifu", "page": "2", "hitsPerPage": "10" }'
```

Example response:

```json
{
 "hits": [
 // … 10 hits
 ],
 "page": 2,
 "hitsPerPage": 10,
 "totalHits": 2100,
 "totalPages": 210
}
```

### Task Management Enhancements
You can now cancel `enqueued` or `processing` tasks using the new `/tasks/cancel` API route. This is useful for interrupting tasks that are taking too long or were mistakenly enqueued.

Example to cancel tasks by `uid`:

```bash
curl \
 -X POST 'http://localhost:7700/tasks/cancel?uids=1,2'
```

### Delete Task History
A new `DELETE` endpoint has been added to the `/tasks` route, allowing you to remove finished tasks from the task history. This helps in managing disk space.

Example to delete tasks by `uid`:

```bash
curl \
 -X DELETE 'http://localhost:7700/tasks?uids=1,2'
```

### Zero Downtime Index Deployment
You can now deploy new versions of indexes without downtime using the `/swap-indexes` API. This allows for seamless updates to your search clients.

Example to swap indexes:

```bash
curl \
 -X POST 'http://localhost:7700/swap-indexes' \
 -H 'Content-Type: application/json' \
 --data-binary '[{ "indexes": ["indexA", "indexB"] }]'
```

### Configuration File Support
You can now use a configuration file to set instance options. By default, Meilisearch looks for a file named `./config.toml`.

Example command to specify a custom config file:

```bash
meilisearch --config-file-path="./my-config.toml"
```

Example configuration file:

```toml
env = "production"
master_key = "MY_MASTER_KEY"
schedule_snapshot = true
```

## Improvements

### Enhanced Filtering for Tasks
New filters are available for the `GET /tasks` endpoint, including `uids`, `canceledBy`, and date fields for better task management.

### Compressed API Requests
Full support for compressed API requests (Gzip, Brotli, Zlib) has been added, improving performance for data transfer.

### Indexing Speed Improvements
Significant improvements have been made to indexing speed, enhancing overall performance.

### Default Bind Address Change
The default bind address has been changed from `127.0.0.1` to `localhost`, making it accessible in both IPv4 and IPv6.

### Increased Max Concurrent Readers
The maximum number of concurrent readers has been increased from 126 to 1024, allowing for better handling of simultaneous requests.

## Other

### Behavior Changes
- The `receivedDocumentIds` field in the task object has been renamed to `providedIds` during the `delete-batch` action.
- A new error `missing_master_key` will be returned when using `/keys` without a master key set, replacing the previous `invalid_api_key` error.
- Some filter parameter names for `/tasks` have been renamed for clarity:
 - `indexUid` to `indexUids`
 - `type` to `types`
 - `status` to `statuses`
