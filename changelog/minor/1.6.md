# v1.6

First released: 2024-01-15

## New Features

### Automated Embeddings Generation for Vector Search
Meilisearch now supports automatic generation of embeddings using either OpenAI or HuggingFace. You can also provide your own embeddings manually.

To configure the embedders, use the following command:

```bash
curl \
 -X PATCH 'http://localhost:7700/indexes/movies/settings' \
 -H 'Content-Type: application/json' \
 --data-binary '{
 "embedders": {
 "default": {
 "source": "openAi",
 "apiKey": "<your-OpenAI-API-key>",
 "model": "text-embedding-ada-002",
 "documentTemplate": "A movie titled \'{{doc.title}}\' whose description starts with {{doc.overview|truncatewords: 20}}"
 },
 "image": {
 "source": "userProvided",
 "dimensions": 512
 },
 "translation": {
 "source": "huggingFace",
 "model": "sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2",
 "documentTemplate": "A movie titled \'{{doc.title}}\' whose description starts with {{doc.overview|truncatewords: 20}}"
 }
 }
 }'
```

### Hybrid Search
This release introduces hybrid search functionality, allowing users to mix keyword and semantic search.

To perform a hybrid search, use the following command:

```bash
curl \
 -X POST 'http://localhost:7700/indexes/movies/search' \
 -H 'Content-Type: application/json' \
 --data-binary '{
 "q": "Plumbers and dinosaurs",
 "hybrid": {
 "semanticRatio": 0.9,
 "embedder": "default"
 }
 }'
```

### Task Queue Webhook
You can now configure a webhook URL that will be called whenever Meilisearch finishes processing a task. Use the following command to set it up:

```bash
./meilisearch \
 --task-webhook-url=https://example.com/example-webhook?foo=bar&number=8 \
 --task-webhook-authorization-header=Bearer aSampleAPISearchKey
```

## Improvements

### Improved Indexing Speed
Indexing performance has been significantly enhanced, reducing the time required to add and update documents. This is achieved by storing and pre-computing less data and only re-indexing necessary data.

### Disk Space Usage Reduction
Meilisearch now consumes less disk space, resulting in smaller database sizes. For example, a ~15Mb dataset can be 40% to 50% smaller.

### Proximity Precision and Performance
You can now customize the accuracy of the proximity ranking rule, which can lead to better performance. Use the following command to set the precision:

```bash
curl \
 -X PATCH 'http://localhost:7700/indexes/books/settings/proximity-precision' \
 -H 'Content-Type: application/json' \
 --data-binary '{
 "proximityPrecision": "byAttribute"
 }'
```

## Other

### Breaking Changes for Vector Search
If you have used vector search between v1.3.0 and v1.5.0, note the following changes:
- When using both `q` and `vector` parameters, you must now include the `hybrid` parameter.
- Defining a model in your embedder settings is now mandatory.
- Vectors should be provided as JSON objects instead of arrays. For example:

```json
"_vectors": { "image2text": [0.0, 0.1, …] } // ✅
"_vectors": [ [0.0, 0.1] ] // ❌
```
