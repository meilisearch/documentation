# v0.25

First released: 2022-01-11

## New Features

### API Key Management
You can now manage and restrict access to API Keys. This feature introduces two default API Keys: `Default Search API Key` and `Default Admin API Key`, which are automatically generated when you first launch MeiliSearch with a `master-key`. 

To create a new API key, use the following endpoint:

```http
POST /keys
```

### Task API Enhancements
The `update` resource has been renamed to `task`, improving clarity in asynchronous updates. You can now access tasks independently of indexes:

```http
GET /tasks
GET /tasks/:taskUid
```

## Improvements

### Enhanced Filtering
The `matches` function now supports numerical values, improving the flexibility of your queries.

### New Error Messages
New error messages have been added for better clarity, including `missing_parameter`, `invalid_api_key_description`, `invalid_api_key_actions`, `invalid_api_key_indexes`, `invalid_api_key_expires_at`, and `api_key_not_found`.

### Improved Error Handling
Error messages related to filter parsing have been enhanced for better user experience.

### Docker Support for ARM
A new MeiliSearch ARM image is now available for Docker, expanding compatibility with different architectures.

### Tokenization Improvements
Improvements have been made to Hangul tokenization and the tokenization process now accurately reflects the size of characters after normalization.

## Other

### Breaking Changes in API Key Management
- The previous `public` and `private` keys have been replaced by the new default keys. If you are upgrading from a version prior to `v0.25.0`, you will need to update your code to use the new keys.
- The `X-MEILI-API-KEY` header has been replaced with the `Authorization: Bearer <apiKey>` header.

### Asynchronous Index Operations
Creating, updating, and deleting indexes are now asynchronous operations, which helps prevent race conditions.

### Compatibility with Dumps
MeiliSearch v0.25 and later are not compatible with dumps created prior to v0.22. To migrate, first load your dump into v0.24.0, create a new dump, and then import it into v0.25.0.

### Docker Image User Change
The Docker image now runs as `root` instead of the `meili` user, reverting to the behavior from versions prior to v0.24.0. This change affects the default location of `data.ms`.
