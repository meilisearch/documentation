# v0.20

First released: 2021-03-30

## Other

### Updated Health Check Response
The `/health` route now returns a `200` status code instead of `204`. This change ensures that the health check is more consistent with standard practices for indicating a healthy service.

### Root Route Response Change
In a production environment, the root route `/` now returns a `200` status code. This update provides clearer feedback about the service status when accessed directly.
