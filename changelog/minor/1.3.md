# v1.3

First released: 2023-07-31

## New Features

### Vector Search — Experimental
You can now use Meilisearch as a vector store, allowing you to add vector embeddings generated by third-party software and use them for searching. This feature can be enabled via the HTTP API.

To enable vector search, send a PATCH request:

```bash
curl \
 -X PATCH 'http://localhost:7700/experimental-features/' \
 -H 'Content-Type: application/json' \
 --data-binary '{
 "vectorStore": true
 }'
```

### Sending Vectorized Documents
Include vector data in your documents using the `_vectors` field. Ensure that all vectors in a dataset have the same size.

```bash
curl -X POST -H 'content-type: application/json' \
 'localhost:7700/indexes/songs/documents' \
 --data-binary '[ 
 { "id": 0, "_vectors": [0, 0.8, -0.2], "title": "Across The Universe" },
 { "id": 1, "_vectors": [1, -0.2, 0], "title": "All Things Must Pass" },
 { "id": 2, "_vectors": [[0.5, 3, 1], [-0.2, 4, 6]], "title": "And Your Bird Can Sing" }
 ]'
```

### Query Meilisearch using Vectors
Use the new `vector` search parameter to find documents with the nearest vector.

```bash
curl -X POST -H 'content-type: application/json' \
'localhost:7700/indexes/songs/search' \
--data-binary '{ "vector": [0, 1, 2] }'
```

### Display Ranking Scores at Search
You can now see the ranking scores for returned documents by using the `showRankingScore` search parameter.

```bash
curl \
 -X POST 'http://localhost:7700/indexes/movies/search' \
 -H 'Content-Type: application/json' \
 --data-binary '{ "q": "Batman Returns", "showRankingScore": true }'
```

### Define Fields to Search on at Search Time
The `attributesToSearchOn` parameter allows you to restrict searches to specific document attributes.

```json
{
  "q": "adventure",
  "attributesToSearchOn": [
    "genre"
  ]
}
```

### Search for Facet Values
The new endpoint `POST /indexes/{index}/facet-search` allows you to search for facet values in filterable attributes.

```bash
curl \
 -X POST 'http://localhost:7700/indexes/movies/facet-search' \
 -H 'Content-Type: application/json' \
 --data-binary '{
 "facetName": "genres",
 "facetQuery": "a"
 }'
```

## Improvements

### Ranking Score Details — Experimental
You can view detailed scores per ranking rule for each document by using the `showRankingScoreDetails` search parameter.

```bash
curl \
 -X POST 'http://localhost:7700/indexes/movies/search' \
 -H 'Content-Type: application/json' \
 --data-binary '{ "q": "Batman Returns", "showRankingScoreDetails": true }'
```

### Sort Facet Values by Count
You can now order facets by count using the `sortFacetValuesBy` property in the index settings.

```bash
curl \
 -X PATCH 'http://localhost:7700/indexes/movies/settings/faceting' \
 -H 'Content-Type: application/json' \
 --data-binary '{
 "sortFacetValuesBy": {"*": "count"}
 }'
```

### Language Support Improvements
Improvements have been made to Japanese word segmentation and separator-based tokenization, enhancing search capabilities.

### Display a Total Count of Tasks
The `/tasks` route now includes a `total` property that displays the total number of tasks in the task queue.

## Other

### Reversion of Filter Escaping Fix
This release reverts a previous fix related to filter escaping that introduced breaking changes. If your filters were working correctly in versions v1.3.3 or v1.3.4, there is no need to upgrade to v1.3.5.
