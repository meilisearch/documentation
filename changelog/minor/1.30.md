# v1.30

First released: 2025-12-15

## New Features

### Network Sharding Modifications
Meilisearch now allows you to modify the number of participants in sharding without needing to restart the document sending process. This is particularly useful for users of the experimental `network` feature who have enabled automatic sharding.

To set up the initial network, you can send a `PATCH` request to the `/network` endpoint:

```http
PATCH /network
```

With the following JSON body:

```json
{
  "self": "ms0",
  "leader": "ms0",
  "remotes": {
    "ms0": {
      "url": "URL_OF_MS0",
      "searchApiKey": "SEARCH_API_KEY_OF_MS0",
      "writeApiKey": "WRITE_API_KEY_OF_MS0"
    },
    "ms1": {
      "url": "URL_OF_MS1",
      "searchApiKey": "SEARCH_API_KEY_OF_MS1",
      "writeApiKey": "WRITE_API_KEY_OF_MS1"
    }
  }
}
```

### New NetworkTopologyChange Tasks
New tasks have been introduced to manage network topology changes. These tasks will handle the rebalancing of documents when adding or removing remotes from the network.

## Improvements

### Enhanced Error Logging
A new warning will be logged when the `FieldidsWeightsMap` is missing an entry, helping to prevent errors during search requests. This improvement enhances the overall user experience by providing clearer feedback.

## Other

### Breaking Changes in Network Feature
Several breaking changes have been introduced for users of the experimental `network` feature:

1. The `Network` object returned or edited by the `/network` route has been modified:
 - The `sharding` boolean has been removed.
 - A `leader` field has been added as an optional string.
 - A `version` field has been added as a UUID.

2. When a `network.leader` is set, certain routes will fail with a `not_a_leader` error if the target's `network.self` does not match the `network.leader`.

3. The `PATCH /network` route no longer returns a `Network` object but instead spawns a new `NetworkTopologyChange` task.

4. Tasks are now duplicated by the leader to the entire network for specific routes, ensuring consistency across all instances.

5. When importing dumps, the `self` and `leader` fields are dropped from the network.

These changes are crucial for users utilizing the network feature, as they affect how tasks and network configurations are managed.
