# v1.33

First released: 2026-01-19

## New Features

### New `/fields` Endpoint
A new POST `/indexes/{indexUid}/fields` endpoint has been added to retrieve detailed metadata about all fields in an index. This includes information on field configuration such as display, search, filtering, and localization settings.

## Improvements

### Enhanced Vector Store Performance
The vector store has been updated to trigger linear scanning even on larger databases. This improvement leads to better performance and result quality, particularly when the number of filtered candidates is small compared to the total number of documents in the index.

### Parallel Cleanup of Old Field IDs
The process for performing dumpless upgrades for instances prior to v1.32.0 has been optimized through multi-threading database fetches. This enhancement significantly reduces the time required for upgrades, from approximately 2 hours and 50 minutes to under 7 minutes.

### Improved Ranking Scores
A fix has been implemented to ensure that all non-blocking buckets are considered for non-final ranking rules. This change enhances the quality of search results, especially when the search cutoff is triggered and vector search is involved.

## Other

### Security Fix for Dump Import
A security vulnerability related to path traversal in the dump import functionality has been addressed. Open-source users who allow importing dumps from untrusted sources are advised to update to v1.33.1 to mitigate this risk. Cloud users do not need to take any action as there is no evidence of exploitation on Meilisearch Cloud.
