# v0.24

First released: 2021-11-16

## New Features

### Enhanced Error Handling
The error handling system has been improved with new error types and a revised structure. The fields `errorCode`, `errorType`, and `errorLink` have been renamed to `code`, `type`, and `link`. New error types include `invalid_request`, `auth`, and `internal`. Failed updates now return a structured error object with the following fields:

```json
{
  "message": "Error message",
  "code": "error_code",
  "type": "error_type",
  "link": "error_link"
}
```

Additionally, several new errors have been introduced, such as `dump_not_found` and `invalid_api_key`, while some existing errors have been renamed or removed.

### Numerical Value Handling in Search Responses
The `attributesToHighlight` feature now supports both numerical and string values. In the search response, numerical values are converted to strings and included in the `_formatted` object, while the original document remains accessible at the root of the response.

## Improvements

### Increased Word Limit per Attribute
The limit of words per attribute has been increased from 1000 to 65535, allowing for more extensive data handling.

### Enhanced Download Script
The `download-latest.sh` script has been improved to include an option for increasing the GitHub rate limit, making it easier to download the latest version without interruptions.

### Reduced Build Size
The overall build size has been optimized, leading to more efficient deployments.

## Other

### Change in Docker User
When using the Docker image, MeiliSearch now runs as the `meili` user instead of `root`. Consequently, the path for storing `data.ms` has changed from `/data.ms` to `/home/meili/data.ms`.
