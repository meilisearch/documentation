# v1.23

First released: 2025-10-13

## New Features

### Index Compaction Task
A new compact route has been introduced for indexes, allowing you to append a compaction task to the queue. This helps reduce fragmentation in the LMDB environment used by Meilisearch, which can lead to significant performance improvements in search and indexing speeds—up to 2-4 times faster.

To initiate a compaction task, you can use the following endpoint:

```http
POST /indexes/{indexUid}/compact
```

### Parallelized Facet Processing
The post-processing of facets has been parallelized, improving the efficiency of bulk facets and word prefix operations. This change reduces the time spent on these operations significantly, with improvements of 4x to 6x observed.

## Improvements

### Enhanced Search Request
A request UID has been added to search routes, allowing for better tracking and management of search requests.

### Performance Enhancements
- The bulk facets and word prefix processing have been optimized for multi-threading, leading to faster operations.
- General improvements in the indexing process have been made to enhance overall performance.
