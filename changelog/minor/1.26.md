# v1.26

First released: 2025-11-17

## New Features

### Custom Metadata for Document Tasks
You can now attach custom metadata to document addition or update tasks, making it easier to track processed documents. This metadata will be displayed when accessing tasks or sent in webhooks.

To use this feature, add the `customMetadata` query parameter to any supported route:

```http
POST /indexes/{indexUid}/documents?customMetadata=my-metadata-for-the-task
```

Supported routes include:

```text
POST /indexes/{indexUid}/documents
PUT /indexes/{indexUid}/documents
DELETE /indexes/{indexUid}/documents/{documentId}
POST /indexes/{indexUid}/documents/delete-batch
POST /indexes/{indexUid}/documents/delete
POST /indexes/{indexUid}/documents/edit
DELETE /indexes/{indexUid}/documents
```

### Support for Modern BERT Models
The `huggingFace` embedder now supports models with the `modernBERT` architecture, allowing you to generate embeddings locally on CPU or GPU. This includes models like Ruri v3 and others.

## Improvements

### Experimental Embedder Failure Modes
You can now choose to ignore certain embedder-related errors, such as:

1. Document template rendering failures
2. Embedding request failures (including missing vectors in `userProvided` embedders)
3. Both types of errors

When errors are ignored, the corresponding documents will not have embeddings, but the associated batch of tasks will not be marked as failed. To enable this feature, OSS users can set the `MEILI_EXPERIMENTAL_CONFIG_EMBEDDER_FAILURE_MODES` environment variable with the desired error types to ignore:

```text
ignore_document_template_failures,ignore_embedder_failures
```

### Experimental Timeout Control for REST Embedders
You can now control the timeout duration for REST embedder requests. OSS users can set the `MEILI_EXPERIMENTAL_REST_EMBEDDER_TIMEOUT_SECONDS` environment variable to a positive integer to specify the timeout duration.
