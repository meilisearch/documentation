# v0.27

First released: 2022-05-09

## New Features

### Nested Field Support
Meilisearch now fully supports nested document fields, allowing you to use nested field locations in any API parameter that accepts document attributes using dot notation. 

For example, when setting your `filterableAttributes`:
```json
{
  "filterableAttributes": [
    "person.firstname"
  ]
}
```
You can filter during the search like this:
```json
{
  "filter": "person.firstname = Gui"
}
```

### Customize Typo Tolerance
You can now customize the typo tolerance settings through new API endpoints. The `typoTolerance` object includes the following properties:
- `enabled`: Whether the typo tolerance feature is enabled (default: `true`).
- `disableOnAttributes`: Disable typo tolerance on specified document attributes (default: ``).
- `disableOnWords`: Disable typo tolerance for specific query terms (default: ``).
- `minWordSizeForTypos`: Customize the minimum size for a word to tolerate typos:
 - `oneTypo`: Minimum size for 1 typo (default: `5`).
 - `twoTypos`: Minimum size for 2 typos (default: `9`).

### Improved Cropping and Highlighting
You can now customize the tags used for highlighting query terms and the crop markers for long field values:
- `highlightPreTag`: Tag before highlighted terms (default: `<em>`).
- `highlightPostTag`: Tag after highlighted terms (default: `</em>`).
- `cropMarker`: Marker around cropped field values (default: `...`).

### Japanese Language Support
New language support for Japanese has been added via the Lindera tokenizer, enhancing the multilingual capabilities of Meilisearch.

### New Instance Options
New options for limiting RAM and CPU usage during indexing:
- `--max-indexing-memory` (`MEILI_MAX_INDEXING_MEMORY`): Sets the maximum RAM size during indexing.
- `--max-indexing-threads` (`MEILI_MAX_INDEXING_THREADS`): Sets the maximum number of threads available for indexing.

## Improvements

### Enhanced Indexation Performance
Indexation time has been improved when inserting documents, making the process faster and more efficient.

### Better Error Messages
Error messages for `sortableAttributes` and `filterableAttributes` during search have been improved for better clarity.

## Other

### Breaking Changes
- The `cropLength` parameter is now defined in terms of the number of words instead of characters, which stabilizes and improves cropping behavior.
- Typo tolerance behavior has changed: a typo on the first letter now counts as 2 typos, which optimizes search performance.
- A hard limit of `1000` retrievable documents has been set during search requests (`GET/POST /indexes/:uid/search`). This limit is a security measure to prevent scraping of the Meilisearch database. You can still retrieve all documents using the `GET /indexes/:uid/documents` route.
- Docker now launches from a different working directory, requiring you to mount your volume in `/meili_data`. Use the following command:
```bash
docker run -it --rm \
 -p 7700:7700 \
 -v $(pwd)/meili_data:/meili_data \
 getmeili/meilisearch:latest
```
- The `meilisearch` binary has been relocated. You can now call it using `meilisearch`, `/meilisearch`, or `/bin/meilisearch`. The following command is correct:
```bash
docker run -it --rm -p 7700:7700 getmeili/meilisearch:latest meilisearch --no-analytics
```
