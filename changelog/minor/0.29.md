# v0.29

First released: 2022-10-03

## New Features

### Customize Matching Strategy
You can now control how Meilisearch matches documents during search by requiring all query terms to be present in the results. This is useful for more precise search results.

To activate this feature, use the following request:

```bash
curl \
 -X POST 'http://localhost:7700/indexes/movies/search' \
 -H 'Content-Type: application/json' \
 --data-binary '{ "q": "american ninja", "matchingStrategy": "all" }'
```

By setting `"matchingStrategy": "all"`, Meilisearch will only return documents containing both "american" and "ninja".

### Thai Language Support
Meilisearch now offers optimized support for the Thai language, enhancing search capabilities for Thai text.

### Filtering Operators
You can now use the `EXISTS` and `IN` operators with the `filter` search parameter. For example, given the following documents:

```json
{ "id": 1, "product": "T-shirt", "price": 20, "color": "yellow" }
{ "id": 2, "product": "T-shirt", "color": "red" }
{ "id": 3, "product": "T-shirt", "price": 30 }
{ "id": 4, "product": "T-shirt", "price": 40 }
```

- The filter `price EXISTS` will select documents `1`, `3`, and `4`.
- The filter `price IN [30, 20]` will select documents `1` and `3`.

### Key Management Enhancements
When managing keys, you can now use the `*` helper to express a set of actions in the `actions` array. For example, instead of specifying each action individually, you can use `indexes.*`.

## Improvements

### Performance Improvements for Indexing
Indexing speed has been improved, particularly for indexes using `filterableAttributes` and during document updates and deletions.

### Auto-batching Enabled by Default
The auto-batching feature is now enabled by default, which should enhance indexing speed. If you previously enabled it manually, you can remove the flag when running Meilisearch v0.29.0.

## Other

### Database Corruption Warning
If you encounter issues after upgrading to v0.29.3, it may indicate that your database is corrupted. In such cases, you will need to re-upload your documents to a fresh index, as using a dump will not resolve the issue.
