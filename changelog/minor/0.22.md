# v0.22

First released: 2021-09-13

## New Features

### Sort at Query Time
You can now sort results at query time. This feature allows for more flexible and dynamic sorting based on your needs. 

If you import your data via a dump, remember to manually add the `sort` ranking rule. You can check your ranking rules using the following endpoint:

```http
GET /indexes/:uid/settings/ranking-rules
```

If the `sort` rule is not listed, you can add it to the 5th position with this request:

```http
POST /indexes/:uid/settings
```

### Windows Compatibility for Download Script
The download script is now compatible with Windows, requiring Cygwin or an equivalent tool for installation.

### Custom Ranking Rules for String Fields
Custom ranking rules now support fields containing strings, expanding the flexibility of your ranking configurations.

## Improvements

### New Syntax for Custom Ranking Rules
The syntax for custom ranking rules has been updated to a more straightforward format. The new syntax is:

```js
"price:asc"
```

The old syntax, which is now deprecated, was:

```js
"asc(price)"
```

### New Indexer Integration
The indexer has been completely rewritten to enhance performance and reliability. This update addresses crashes and RAM consumption issues, increases indexing speed, and resolves indexing bugs. 

Please note that using the new indexer requires re-indexation of all your documents. This will be handled automatically when migrating your data from a previous version.

## Other

### Breaking Changes
- The new indexer requires re-indexation of all documents. For more information on updating to the latest version of MeiliSearch, refer to the dedicated guide on updating.
