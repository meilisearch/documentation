# v0.26

First released: 2022-03-14

## New Features

### Tenant Tokens
You can now use tenant tokens to restrict access to documents based on the end-user making the search request. This feature enables multi-tenant indexes, allowing for more secure and tailored search experiences.

A tenant token is a JSON Web Token (JWT) generated by combining an API key with specific permissions and rules for the end user's search.

For more information on tenant tokens, refer to the dedicated guide in the documentation.

### Auto-Batching (Experimental)
The new auto-batching feature improves indexing speed by automatically batching together consecutive document additions. This feature is currently experimental and may be modified or removed based on user feedback.

To enable auto-batching, use the following command:

```bash
meilisearch --enable-auto-batching
```

## Improvements

### Allow Payloads with No Documents
You can now send payloads that do not contain any documents, providing more flexibility in your API interactions.

### Accept `null` Values for Numeric Fields on CSV Upload
When uploading CSV files, you can now include `null` values for numeric fields, enhancing data handling capabilities.

### Update Dashboard for Rebranding
The dashboard has been updated to reflect the new branding, ensuring a consistent user experience.

## Other

### New Dump Behavior
Importing a dump into an instance with an existing database using the `--import-dump` option will now throw an error. New flags `--ignore-dump-if-db-exists` and `--ignore-missing-dump` have been added for better control over dump imports.

### New Flag Behavior for the CLI
The behavior of several CLI options has changed. For example, the `--no-analytics` option should now be used without a value:

```bash
meilisearch --no-analytics
```

### Removal of armv8 Binary
The armv8 binary has been removed from the release assets. Users should now use the aarch64 binary instead.
