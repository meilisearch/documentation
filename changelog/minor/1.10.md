# v1.10

First released: 2024-08-26

## New Features

### Federated Search
You can now perform multi-search requests and receive a single list of results. Use the new `federation` setting in the `/multi-search` route:

```bash
curl \
 -X POST 'http://localhost:7700/multi-search' \
 -H 'Content-Type: application/json' \
 --data-binary '{
 "federation": {
 "offset": 5,
 "limit": 10
 },
 "queries": [
 {
 "q": "Batman",
 "indexUid": "movies"
 },
 {
 "q": "Batman",
 "indexUid": "comics"
 }
 ]
 }'
```

### Experimental: `CONTAINS` Filter Operator
You can now use the `CONTAINS` filter operator to find partial matches in your search queries. First, enable the feature:

```bash
curl \
 -X PATCH 'http://localhost:7700/experimental-features/' \
 -H 'Content-Type: application/json' \
 --data-binary '{
 "containsFilter": true
 }'
```

Then, use it in your search:

```bash
curl \
 -X POST 'http://localhost:7700/indexes/movies/search' \
 -H 'Content-Type: application/json' \
 --data-binary '{
 "q": "super hero",
 "filter": "synopsis CONTAINS spider"
 }'
```

### Language Settings
You can now explicitly set the languages used in document fields and search queries using the `localizedAttributes` index setting:

```bash
curl \
 -X PATCH 'http://localhost:7700/indexes/movies/settings' \
 -H 'Content-Type: application/json' \
 --data-binary '{
 "localizedAttributes": [
 {"locales": ["jpn"], "attributePatterns": ["*_ja"]},
 {"locales": ["eng"], "attributePatterns": ["*_en"]}
 ]
 }'
```

### Experimental: Edit Documents with a Rhai Function
You can now edit documents directly from Meilisearch using a Rhai function. First, enable the feature:

```bash
curl \
 -X PATCH 'http://localhost:7700/experimental-features/' \
 -H 'Content-Type: application/json' \
 --data-binary '{
 "editDocumentsByFunction": true
 }'
```

Then, use the `/documents/edit` route:

```bash
curl http://localhost:7700/indexes/movies/documents/edit \
 -H 'content-type: application/json' \
 -d '{
 "function": "doc.title = `✨ ${doc.title.to_upper} ✨`",
 "filter": "id > 3000"
 }'
```

## Improvements

### Search Improvements with CLI Flags
You can now configure the search engine's behavior with two experimental CLI flags:
- Set the number of concurrent searches per core with `--experimental-nb-searches-per-core`.
- Define how many seconds before a search is considered irrelevant with `--experimental-drop-search-after`.

### Better Search Handling Under Heavy Loads
Improvements have been made to enhance search performance under heavy loads, including optimizations to the search queue and index scheduler.

### Speed Improvements for Document Deletion
The autobatching of document deletions by filter has been improved, which helps to reduce task queue congestion.

## Other

### Breaking Changes: REST API Parameter Changes
Several parameters in the REST API have been removed or replaced to simplify the configuration. The following parameters have been removed:
- `query`, `inputField`, `inputType`, `pathToEmbeddings`, and `embeddingObject`.

They have been replaced with:
- `request`: A JSON value representing the request made by Meilisearch to the remote embedder.
- `response`: A JSON value representing a fragment of the response made by the remote embedder to Meilisearch.

### Important Change: Minimum Ubuntu Version
Meilisearch is now only compatible with Ubuntu 20.04 and later due to changes in the GitHub Actions runner.
