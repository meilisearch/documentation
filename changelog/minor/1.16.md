# v1.16

First released: 2025-08-04

## New Features

### Multimodal Embeddings
You can now index and search non-textual documents, including images, using multimodal embeddings. This feature allows for AI-powered searches that provide a common semantic representation for images, texts, and other data types.

To enable the `multimodal` experimental feature, use the following command:

```bash
curl \
 -X PATCH 'MEILISEARCH_URL/experimental-features/' \
 -H 'Content-Type: application/json' \
 --data-binary '{
 "multimodal": true
 }'
```

Next, configure an embedder provider that supports multimodal embeddings, such as VoyageAI:

```bash
curl \
 -X PATCH 'MEILISEARCH_URL/indexes/INDEX_NAME/settings/embedders' \
 -H 'Content-Type: application/json' \
 --data-binary '{
 "voyage": {
 "source": "rest",
 "url": "https://api.voyageai.com/v1/multimodalembeddings",
 "apiKey": "VOYAGE_API_KEY",
 "indexingFragments": {
 "text": {
 "value": {
 "content": [
 {
 "type": "text",
 "text": "A movie titled {{doc.title}} whose description starts with {{doc.overview|truncateWords:20}}."
 }
 ]
 }
 },
 "poster": {
 "value": {
 "content": [
 {
 "type": "image_url",
 "image_url": "{{doc.poster}}"
 }
 ]
 }
 }
 },
 "searchFragments": {
 "poster": {
 "value": {
 "content": [
 {
 "type": "image_url",
 "image_url": "{{media.poster}}"
 }
 ]
 }
 },
 "image": {
 "value": {
 "content": [
 {
 "type": "image_base64",
 "image_base64": "data:{{media.image.mime}};base64,{{media.image.data}}"
 }
 ]
 }
 },
 "text": {
 "value": {
 "content": [
 {
 "type": "text",
 "text": "{{q}}"
 }
 ]
 }
 }
 },
 "request": {
 "inputs": [
 "{{fragment}}",
 "{{..}}"
 ],
 "model": "voyage-multimodal-3"
 },
 "response": {
 "data": [
 {
 "embedding": "{{embedding}}"
 },
 "{{..}}"
 ]
 }
 }
 }'
```

To perform a search using an image, use the following command:

```bash
curl -X POST 'MEILISEARCH_URL/indexes/INDEX_NAME/search' \
 -H 'content-type: application/json' \
 --data-binary '{
 "media": {
 "poster": "https://image.tmdb.org/t/p/w500/pgqj7QoBPWFLLKtLEpPmFYFRMgB.jpg"
 },
 "hybrid": {
 "embedder": "EMBEDDER_NAME"
 }
 }'
```

You can also perform a text search:

```bash
curl -X POST 'MEILISEARCH_URL/indexes/INDEX_NAME/search' \
 -H 'content-type: application/json' \
 --data-binary '{
 "q": "A movie with lightsabers in space",
 "hybrid": {
 "embedder": "voyage",
 "semanticRatio": 0.5
 }
 }'
```

### New `/export` Route
A new `/export` route has been introduced to facilitate the transfer of documents between instances without needing to create a dump or snapshot. This is particularly useful for migrating from a local instance to Meilisearch Cloud.

To transfer data, use the following command:

```bash
curl \
 -X POST 'MEILISEARCH_URL/export' \
 -H 'Content-Type: application/json' \
 --data-binary '{
 "url": "http://localhost:7711"
 }'
```

This command will initiate the export and start migrating data. If necessary, you may need to provide an API key with full admin permissions.

## Improvements

### Nested Wildcards Support
Support for nested wildcards in `attributes_to_search_on` has been added, enhancing search flexibility.

### Geo Field Extraction
Improvements have been made to the extraction of geo fields from documents, allowing for better handling of geographical data.

### CPU Utilization for Importing Dumps
Meilisearch now utilizes all available CPUs when importing dumps, improving performance during this operation.

### Live Embedder Error Display
The last embedder error is now displayed live in batches, making it easier to troubleshoot issues.

### Read-Only Admin Key
A Read-Only Admin key will be created in new empty databases to prevent accidental writes while investigating.

## Other

### Breaking Changes
- The introduction of the new `/export` route may require adjustments in how data is managed between instances.
- A fallback instance option has been added to revert to the old indexer, which may affect existing configurations.
