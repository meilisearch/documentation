# v0.11

First released: 2020-06-15

## New Features

### Faceted Search Support
You can now perform faceted searches, allowing you to filter results based on specific attributes. This enhances your search capabilities by enabling users to narrow down results effectively.

### Support for Facet Count
The new facet count feature allows you to retrieve the number of documents for each facet value, providing better insights into your data distribution.

### SSL Support
SSL support has been added, enabling secure connections to your Meilisearch instance. This is crucial for protecting data in transit.

### Telemetry
Telemetry features have been introduced to help you monitor the performance and usage of your Meilisearch instance, providing valuable insights for optimization.

## Improvements

### Enhanced Settings Consistency
Improvements have been made to ensure better consistency across settings, leading to a more reliable configuration experience.

### Max Payload Size Override
You can now override the maximum payload size, giving you more control over the data you can send in requests.

### Better Terminal Greeting
The terminal greeting has been improved for a more user-friendly experience when starting the application.

### Improved Error Handling
When a deserialization error occurs, the error message from Serde will now be returned, making it easier to diagnose issues.

### Dashboard Accessibility
The NormalizePath middleware has been fixed to ensure that the dashboard is accessible without issues.

### Dockerfile Enhancements
An exposed port has been added to the Dockerfile, simplifying the process of running Meilisearch in a containerized environment.

## Other

### HTTP Framework Change
The HTTP framework has been changed from `tide` to `actix-web`, which may affect how you interact with the API. Ensure to test your integrations accordingly.

### Field Renaming
The field `fieldsFrequency` has been renamed to `fieldsDistribution` in stats, which may require updates to any scripts or applications relying on the previous field name.
