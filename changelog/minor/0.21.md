# v0.21

First released: 2021-08-23

## New Features

### New Web UI
A completely redesigned web interface is now available, providing a more user-friendly experience for managing your MeiliSearch instance.

### Support for Facets with Numbers
You can now use numeric values in facets, enhancing your ability to filter and categorize search results.

### Enhanced Logging Capabilities
Logging has been improved with the introduction of the `MEILI_LOG_LEVEL` environment variable and `--log-level` option. The debug level now outputs detailed query and search results.

### Unified Analytics Management
Analytics tools (Amplitude and Sentry) are now unified, allowing you to disable them simultaneously using the `MEILI_NO_ANALYTICS` environment variable and `--no-analytics` flag.

### New Update Status
A new status, `processing`, has been added to provide better visibility into the status of your operations.

### Additional Fields in Dump Status
The `/dumps/{dump_id}/status` route now includes `startedAt` and `finishedAt` fields, giving you more information about the dump process.

## Improvements

### Search Performance Enhancements
Overall search performance has been improved, making queries faster and more efficient.

### Highlighting Matches in Arrays and Objects
The `attributesToHighlight` feature now highlights matches found within arrays and objects, improving the visibility of relevant search results.

### Phrase Search Support
You can now perform phrase searches by enclosing your search terms in quotes, allowing for more precise queries.

### Reduced RAM Usage During Indexation
Memory usage during the indexing process has been optimized, leading to better resource management.

### Increased Index Limit
The previous limit of 200 indexes per database has been removed, allowing for greater flexibility in managing your data.

### Improved Exactness Ranking Rule
The `exactness` ranking rule has been enhanced to provide more relevant search results.

### Consistent Cropping
Cropping behavior has been made consistent across different use cases, ensuring uniformity in how results are displayed.

### Payload Limit Size Specification
Users can now specify the payload limit size unit (e.g., Kb, Mb, Gb), providing more control over data handling.

### Improved `/stats` Route Speed
The performance of the `/stats` route has been improved, resulting in faster response times.

### Compilation Time Optimization
Compilation time has been reduced through the implementation of a caching strategy.

## Other

### Breaking Changes
- The index setting `attributesForFaceting` has been renamed to `filterableAttributes`.
- The `filters` and `facetsFilter` search parameters have been replaced by the `filter` parameter. To use the `filter` parameter, you must add the attribute you want to filter on to the `filterableAttributes` index setting. Use the `=` syntax (e.g., `genres = comedy`) for filter expressions, as the `:` syntax is deprecated. Filter expressions can be combined as strings or nested arrays:
 - String: 
 ```json
{
  "filter": "genre = comedy AND price > 13"
}
```
 - Array: 
 ```json
{
  "filter": [
    "genre = comedy",
    "price > 13"
  ]
}
```
- When using the `filter` parameter on an array, the operator `!=` will only return results that do not contain the specified value.
- The new default ranking rules are:
 ```json
[
  "words",
  "typo",
  "proximity",
  "attribute",
  "exactness"
]
```
- The `fieldsDistribution` setting has been renamed to `fieldDistribution`.
- The `MAX_MDB_SIZE` environment variable and `--max-mdb-size` option flag have been renamed to `MAX_INDEX_SIZE` and `--max-index-size`.
- In the `/version` route, `buildDate` is now named `commitDate`, and the format has changed to ensure MeiliSearch is reproducible.
