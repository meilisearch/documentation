# v0.28

First released: 2022-07-11

## New Features

### Smart Crop
The smart crop feature enhances the existing cropping algorithm by identifying the window(s) with the highest density of query terms within the specified `cropLength`. This ensures that the most relevant parts of the text are highlighted.

### Deterministic API Key Management
You can now create deterministic `key` values by specifying a `uid` field during creation. The `uid` must be a UUID v4. If omitted, Meilisearch generates it automatically. The `key` is generated by hashing the master key and the `uid`:

```bash
echo -n $HYPHENATED_UUID | openssl dgst -sha256 -hmac $MASTER_KEY
```

This ensures that `key` values are consistent across instances with the same configuration. Additionally, a `name` field has been added for easier API key retrieval, and new actions for managing API keys have been introduced.

### Hebrew Language Support
Support for the Hebrew language has been added, utilizing the Niqqud library for improved text processing.

### Improved Tokenizer Contribution Experience
The process for contributing new languages to the Meilisearch tokenizer has been significantly enhanced. Users can now follow the CONTRIBUTING.md file to integrate their intended normalizer/segmenter.

### Customizable Limits for Facet Values
When searching with the `facets` parameter, Meilisearch now returns a maximum of 100 different facet values per attribute, which can be customized. To increase this limit to 3000:

```bash
curl \
 -X PATCH 'http://localhost:7700/indexes/books/settings/faceting' \
 -H 'Content-Type: application/json' \
 --data-binary '{
 "maxValuesPerFacet": 3000
 }'
```

### Customizable Limits for Documents Returned by Search
The maximum number of documents returned by a search request is now customizable. To increase this limit to 2000:

```bash
curl \
 -X PATCH 'http://localhost:7700/indexes/books/settings/pagination' \
 -H 'Content-Type: application/json' \
 --data-binary '{
 "maxTotalHits": 2000
 }'
```

## Improvements

### Enhanced Pagination for Indexes and Documents
The response for getting the list of indexes and documents now includes pagination. You can use `offset` and `limit` parameters to navigate through results, and the response will include `results`, `limit`, `offset`, and `total` fields.

### Changes to Search Endpoint Parameters
Several parameters in the `/indexes/{uid}/search` endpoint have been renamed for clarity:
- `nbHits` is now `estimatedTotalHits`
- `facetsDistribution` is now `facets`
- `matches` is now `showMatchesPosition`

### Improved Task Filtering
The `GET /tasks` endpoint now supports filtering by `type`, `status`, and `indexUid`, allowing for more precise task management.

## Other

### Breaking Changes in API
This release includes several breaking changes, particularly in endpoint names and response fields. For example:
- The endpoint to update an index is now `PATCH /indexes/{indexUid}` instead of `PUT`.
- The `attributesToRetrieve` parameter has been renamed to `fields` in the documents endpoints.

### Changes to API Key Management
The `apiKeyPrefix` has been renamed to `apiKeyUid`, and it is no longer possible to update certain fields of an API key after creation. Additionally, the response for getting the keys list is now paginated.

### Changes to Dumps
Dump creation is now an asynchronous task, and the endpoint for checking dump status has been removed. You can now check the status of dump tasks using the `GET /tasks?type=dumpCreation` endpoint.
