# v1.1

First released: 2023-04-03

## New Features

### Multi-index Search
You can now perform searches across multiple indexes in a single HTTP request using the new `/multi-search` endpoint:

```bash
curl \
 -X POST 'http://localhost:7700/multi-search' \
 -H 'Content-Type: application/json' \
 --data-binary '{
 "queries": [
 {
 "indexUid": "products",
 "q": "Nike",
 "limit": 1
 },
 {
 "indexUid": "brands",
 "q": "Nike",
 "limit": 1
 }
 ]
 }'
```

The response will include results for each queried index:

```json
{
 "results": [
 {
 "indexUid": "products",
 "hits": […],
 "query": "Nike",
 "processingTimeMs": 1,
 "limit": 1,
 "offset": 0,
 "estimatedTotalHits": 17
 },
 {
 "indexUid": "brands",
 "hits": […],
 "query": "Nike",
 "processingTimeMs": 0,
 "limit": 1,
 "offset": 0,
 "estimatedTotalHits": 7
 }
 ]
}
```

### Introduce `facetStats`
Queries with the `facets` parameter now return a `facetStats` object by default, providing minimum and maximum values for numerical facets:

```bash
curl \
 -X POST 'http://localhost:7700/indexes/movies/search' \
 -H 'Content-Type: application/json' \
 --data-binary '{
 "facets": ["price"]
 }'
```

Example response:

```json
{
 "hits": […],
 "facetDistribution": {
 "price": {…}
 },
 "facetStats": {
 "price": {
 "min": 2,
 "max": 60
 }
 }
}
```

### New Geosearch Filter: `_geoBoundingBox`
You can now filter search results based on geographic boundaries using the `_geoBoundingBox` filter:

```bash
curl \
 -X POST 'http://localhost:7700/indexes/restaurants/search' \
 -H 'Content-type:application/json' \
 --data-binary '{ "filter": "_geoBoundingBox([45.472735, 9.184019],[45.473711, 9.185613] )" }'
```

### Unlimited Index Size and Count
Meilisearch now allows you to create an unlimited number of indexes, with size limits dictated only by your operating system's memory address space.

### Experimental Feature: Prometheus Monitoring
An experimental feature is now available for monitoring metrics with Prometheus. Enable it by launching Meilisearch with the following flag:

```bash
meilisearch --experimental-enable-metrics
```

The `/metrics` route will be available for Prometheus integration.

## Improvements

### Enhanced Language Support
- Improved Greek support by normalizing diacritics and final sigma.
- Enhanced Arabic support by ignoring Tatweel.
- Language detection now occurs during indexing, reducing errors during search.

### Customizable CSV Delimiter
You can now customize the CSV delimiter (default is `,`) using the `csvDelimiter` parameter when adding or updating documents.

### Improved Error Messages
Error messages now include "did you mean ...?" suggestions for common typos in Meilisearch parameters.

### Automatic Task Batching
Addition and deletion tasks are now automatically batched together to speed up indexing.

### Cache Index Stats Results
Index statistics results are now cached for improved performance.

## Other

### Breaking Changes
- The auto-batching of additions and deletions has been disabled to address issues with task queue management.
- If you encountered issues with version v1.1.0, it may indicate a corrupted database. Users are advised to recreate their indexes and re-upload documents, as using a dump will not resolve the issue.
