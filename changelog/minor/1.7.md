# v1.7

First released: 2024-03-11

## New Features

### Improved AI-powered Search — Experimental
You can now activate AI-powered search by setting `vectorStore` to `true` in the `/experimental-features` route. This feature enhances search capabilities using embeddings.

### New OpenAI Embedding Models
Two new models are available for configuring OpenAI embedders:
- `text-embedding-3-small` with a default dimension of 1536.
- `text-embedding-3-large` with a default dimension of 3072.

These models improve search result relevancy and are more cost-effective.

### Custom OpenAI Model Dimensions
You can now specify custom dimensions for the new OpenAI models. Ensure dimensions are greater than 0 and within the model size limits:

```json
"embedders": {
 "new_model": {
 "source": "openAi",
 "model": "text-embedding-3-large",
 "dimensions": 512
 },
 "legacy_model": {
 "source": "openAi",
 "model": "text-embedding-ada-002"
 }
}
```

### GPU Support for Hugging Face Embeddings
You can enable CUDA to utilize Nvidia GPUs for faster embedding generation. Follow these steps to activate GPU support:

1. Install CUDA dependencies.
2. Clone and compile Meilisearch with the `cuda` feature:
 ```bash
 cargo build --release --package meilisearch --features cuda
 ```
3. Launch the compiled Meilisearch binary.

## Improvements

### Stabilized `showRankingScoreDetails`
The `showRankingScoreDetails` parameter is now stable. Use it in the `/search` endpoint to view detailed scores per ranking rule for each returned document:

```bash
curl \
 -X POST 'http://localhost:7700/indexes/movies/search' \
 -H 'Content-Type: application/json' \
 --data-binary '{ "q": "Batman Returns", "showRankingScoreDetails": true }'
```

The response will include a `_rankingScoreDetails` field with detailed scoring information.

### Improved Indexing Speed and Reduced Memory Crashes
Several enhancements have been made to improve indexing speed and reduce memory usage, including:
- Auto-batch task deletion.
- Significant speed improvements for vector store indexing.
- Capping maximum memory of grenad sorters.

### Improved Logging
Log messages now follow a new format for better readability:

```bash
# new format
2024-02-06T14:54:11Z INFO actix_server::builder: 200: starting 10 workers
```

You can also configure Meilisearch to output logs in JSON format by using the `--experimental-logs-mode json` option when launching the instance.

### New Experimental API Routes
Two new experimental API routes have been introduced:
- `/logs/stream`: Stream logs with configurable log levels.
- `/logs/stderr`: Configure default log output for non-stream logs.

## Other

### Security Update
A security flaw in the implementation of HTTP/2 has been addressed to prevent potential slowdowns of your instance. The web stack has been updated to the latest version containing this fix.

### Fix for Nested Attributes Reindexing
A regression that prevented reindexing when adding sortable and filterable nested attributes has been fixed.
