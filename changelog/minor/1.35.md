# v1.35

First released: 2026-02-02

## New Features

### Performance Details in Search Responses
You can now gain insights into the performance of your search queries. By adding the `showPerformanceDetails` parameter to your search requests, you will receive detailed performance metrics in the response.

#### Search Request Example
```http
POST /indexes/<index_uid>/search
```
```json
{
  "q": "glass",
  "showPerformanceDetails": true
}
```

#### Search Response Example
```json
{
 "hits": [
 // hits ..
 ],
 "query": "glass",
 "processingTimeMs": 5,
 "limit": 20,
 "offset": 0,
 "estimatedTotalHits": 1,
 "requestUid": "<uuid>",
 "performanceDetails": {
 "wait for permit": "295.29µs",
 "search > tokenize": "436.67µs",
 "search > resolve universe": "649.00µs",
 "search > keyword search": "515.71µs",
 "search > format": "288.54µs",
 "search": "3.56ms"
 }
}
```

### Performance Details in Multi-Search
The `showPerformanceDetails` parameter is also available in the `POST /multi-search` endpoint, allowing you to analyze the performance of multiple search queries at once.

#### Multi-Search Request Example
```http
POST /multi-search
```
```json
{
  "queries": [
    {
      "indexUid": "<index_uid>",
      "q": "glass",
      "showPerformanceDetails": true
    }
  ]
}
```

#### Multi-Search Response Example
```json
{
 "results": [
 {
 "indexUid": "<index_uid>",
 "hits": [
 // hits ..
 ],
 "query": "glass",
 "processingTimeMs": 5,
 "limit": 20,
 "offset": 0,
 "estimatedTotalHits": 1,
 "requestUid": "<uuid>",
 "performanceDetails": {
 "wait for permit": "295.29µs",
 "search > tokenize": "436.67µs",
 "search > resolve universe": "649.00µs",
 "search > keyword search": "515.71µs",
 "search > format": "288.54µs",
 "search": "3.56ms"
 }
 }
 ]
}
```

### Performance Details in Similar Queries
You can also include the `showPerformanceDetails` parameter in requests to the `POST /indexes/<index_uid>/similar` and `GET /indexes/<index_uid>/similar` endpoints.

#### Similar Request Example
```http
POST /indexes/<index_uid>/similar
```
```json
{
  "id": 143,
  "embedder": "manual",
  "showPerformanceDetails": true
}
```

#### Similar Response Example
```json
{
 "hits": [
 // hits ..
 ],
 "id": "143",
 "processingTimeMs": "[duration]",
 "limit": 20,
 "offset": 0,
 "estimatedTotalHits": 4,
 "performanceDetails": {
 "search > format": "244.92µs",
 "search": "1.25ms"
 }
}
```

## Improvements

### Enhanced Indexing Performance
The multithreaded post-processing of facets and prefixes is now always enabled, resulting in faster indexing on multi-core machines. This improvement benefits cloud users upgrading to v1.35.0.

## Other

### Breaking Change: Updated Response for Fields Route
The `POST /indexes/<index_uid>/fields` route now returns a structured object instead of an array. The new response includes `results`, `offset`, `limit`, and `total` fields, providing better clarity on the number of fields matching a given filter.

#### Example: Before and After
**Before:**
```json
[
 // fields ..
]
```

**After:**
```json
{
 "results": [
 // [fields .. ]
 ],
 "offset": 0,
 "limit": 20,
 "total": 0
}
```
