# v1.13

First released: 2025-02-17

## New Features

### AI-Powered Search Integration
AI-powered search is now available by default for all users, enhancing search capabilities without requiring manual activation.

### Experimental Feature: Dumpless Upgrades
You can now upgrade to new Meilisearch releases without generating a dump. Use the following command after updating the Meilisearch binary:

```bash
./meilisearch --experimental-dumpless-upgrade
```

This process is faster and more efficient, but it is recommended to create a backup snapshot before migrating.

### Experimental Feature: Remote Federated Search Requests
You can perform remote federated search requests across multiple Meilisearch instances. First, enable the `network` feature:

```bash
curl \
 -X PATCH 'MEILISEARCH_URL/experimental-features/' \
 -H 'Content-Type: application/json' \
 --data-binary '{
 "network": true
 }'
```

Then, configure your network of instances:

```bash
curl \
 -X PATCH 'MEILISEARCH_URL/network/' \
 -H 'Content-Type: application/json' \
 --data-binary '{
 "remotes": {
 "ms-0": {
 "url": "http://ms-1235.example.meilisearch.io",
 "searchApiKey": "Ecd1SDDi4pqdJD6qYLxD3y7VZAEb4d9j6LJgt4d6xas"
 },
 "ms-1": {
 "url": "http://ms-4242.example.meilisearch.io",
 "searchApiKey": "hrVu-OMcjPGElK7692K7bwriBoGyHXTMvB5NmZkMKqQ"
 }
 },
 "self": "ms-0"
 }'
```

Finally, make a `/multi-search` query:

```bash
curl \
 -X PATCH 'MEILISEARCH_URL/multi-search/' \
 -H 'Content-Type: application/json' \
 --data-binary '{
 "federation": {},
 "queries": [
 {
 "q": "Batman returns dark",
 "indexUid": "movies",
 "federationOptions": {
 "remote": "ms-0"
 }
 },
 {
 "q": "Batman returns dark",
 "indexUid": "movies",
 "federationOptions": {
 "remote": "ms-1"
 }
 }
 ]
}'
```

## Improvements

### Enhanced Task Auto-Batching
The task auto-batching process has been improved to allow users to limit the total size of a batch.

### Prometheus Metrics for Task Queue Latency
New Prometheus metrics have been added to measure task queue latency, providing better insights into performance.

### Faster Index Listing
The request to list indexes has been optimized for improved speed.

### New `usedDatabaseSize` Field
A new `usedDatabaseSize` field has been added to the `/stats` route, giving users more information about database usage.

### Embeddings Info on `/stats` Route
The `/stats` route now exposes information about embeddings, enhancing monitoring capabilities.

## Other

### Breaking Changes
- The `vectorStore` value is no longer accepted for the `/experimental-features` route.
- Ollama URLs must now end with either `/api/embed` or `/api/embeddings`.
- Modified error codes:
 - `invalid_embedder` is now split into `invalid_search_embedder` and `invalid_similar_embedder`.
 - `invalid_hybrid_query` has been renamed to `invalid_search_hybrid_query`.
