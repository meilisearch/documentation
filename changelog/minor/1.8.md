# v1.8

First released: 2024-05-06

## New Features

### Hybrid Search Enhancements
This release introduces significant updates to hybrid search, including a new `distribution` embedder setting and support for new embedder sources. These changes aim to improve the ranking of search results.

### Negative Keywords
You can now exclude specific terms from your search queries using the `-` operator. For example:

```bash
curl \
 -X POST 'http://localhost:7700/indexes/places/search' \
 -H 'Content-Type: application/json' \
 --data-binary '{ "q": "-escape room" }'
```

This will return documents containing `room` but not `escape`.

### New Embedders: Ollama and Generic REST Embedder
You can now use the Ollama model and any RESTful interface for embeddings. Here’s how to configure the Ollama embedder:

```json
{
  "default": {
    "source": "ollama",
    "url": "http://localhost:11434/api/embeddings",
    "apiKey": "<foobarbaz>",
    "model": "nomic-embed-text",
    "documentTemplate": "A document titled '{{doc.title}}' whose description starts with {{doc.overview|truncatewords: 20}}"
  }
}
```

For a generic REST embedder, use:

```json
{
  "default": {
    "source": "rest",
    "url": "http://localhost:12345/api/v1/embed",
    "apiKey": "187HFLDH97CNHN",
    "dimensions": 512,
    "documentTemplate": "A document titled '{{doc.title}}' whose description starts with {{doc.overview|truncatewords: 20}}",
    "inputField": [
      "data",
      "text"
    ],
    "inputType": "text",
    "query": {
      "model": "MODEL_NAME",
      "dimensions": 512
    },
    "pathToEmbeddings": [
      "data"
    ],
    "embeddingObject": [
      "embedding"
    ]
  }
}
```

## Improvements

### Faster Upgrade Path
This version allows for an upgrade from v1.8.x to v1.9.0 without needing to regenerate embeddings. Follow these steps:
1. Upgrade to v1.8.4 by restarting Meilisearch on your v1.8.x database.
2. Create a dump.
3. Import the dump into Meilisearch v1.9.

### Search Robustness Updates
- **Search Cutoff**: Requests taking longer than 1500ms will be interrupted. You can customize this value using the `/settings` endpoint:

```bash
curl \
 -X PATCH 'http://localhost:7700/indexes/movies/settings' \
 -H 'Content-Type: application/json' \
 --data-binary '{
 "searchCutoffMs": 150
 }'
```

- **Concurrent Search Request Limits**: A limit for concurrent search requests has been introduced to prevent excessive RAM usage. Relaunch your instance with:

```bash
./meilisearch --experimental-search-queue-size 100
```

## Other

### Breaking Changes: `_semanticScore`
To enhance performance, the following fields will no longer be included in search responses:
- The `vector` field
- The `_semanticScore` field (use `_rankingScore` instead)
- The query vector and its value when `"showRankingScoreDetails": true` is set

These changes are aimed at improving search response times and reducing bandwidth usage.
