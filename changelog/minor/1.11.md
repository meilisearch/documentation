# v1.11

First released: 2024-10-28

## New Features

### AI-Powered Search Improvements
This release introduces significant enhancements to AI-powered search, marking the first step towards stabilization. Key changes include:

- The `hybrid.embedder` parameter is now mandatory for both `GET` and `POST` requests to `/indexes/{:indexUid}/search`.
- The `embedder` parameter is also mandatory for `GET` and `POST` requests to `/indexes/{:indexUid}/similar`.
- Queries that include `vector` but not `q` will now default to a pure semantic search.

### Federated Search Enhancements
New options for federated searches have been added:

- **Facet Distribution by Index**: Use the `facetsByIndex` parameter to get facet stats for each index in a multi-search request.

```json
POST /multi-search
{
 "federation": {
 "limit": 20,
 "offset": 0,
 "facetsByIndex": {
 "movies": ["title", "id"],
 "comics": ["title"]
 }
 },
 "queries": [
 {
 "q": "Batman",
 "indexUid": "movies"
 },
 {
 "q": "Batman",
 "indexUid": "comics"
 }
 ]
}
```

- **Merged Facet Information**: Combine facet stats from multiple indexes using `mergeFacets`.

```json
POST /multi-search
{
 "federation": {
 "limit": 20,
 "offset": 0,
 "facetsByIndex": {
 "movies": ["title", "id"],
 "comics": ["title"]
 },
 "mergeFacets": {
 "maxValuesPerFacet": 10
 }
 },
 "queries": [
 {
 "q": "Batman",
 "indexUid": "movies"
 },
 {
 "q": "Batman",
 "indexUid": "comics"
 }
 ]
}
```

### Experimental `STARTS WITH` Filter Operator
You can now enable the `STARTS WITH` filter operator for more flexible querying.

```bash
curl \
 -X PATCH 'http://localhost:7700/experimental-features/' \
 -H 'Content-Type: application/json' \
 --data-binary '{
 "containsFilter": true
 }'
```

Use it in your search queries:

```json
curl \
 -X POST http://localhost:7700/indexes/movies/search \
 -H 'Content-Type: application/json' \
 --data-binary '{
 "filter": "hero STARTS WITH spider"
 }'
```

## Improvements

### Timeout Settings for Operations
Timeouts have been added for read and write operations, enhancing reliability during high-load scenarios.

### Enhanced Embedding Request Timeout
A 3-second timeout has been introduced for embedding requests made during searches, improving overall search performance.

### Vector Database Indexing Performance
The new `binaryQuantized` option can be enabled to improve indexing performance significantly, especially for large datasets.

```bash
curl \
 -X PATCH 'http://localhost:7700/indexes/movies/settings' \
 -H 'Content-Type: application/json' \
 --data-binary '{
 "embedders": {
 "image2text": {
 "binaryQuantized": true
 }
 }
 }'
```

## Other

### Breaking Changes
- The `hybrid.embedder` and `embedder` parameters are now mandatory for specific search and similarity requests.
- The `semanticRatio` parameter is ignored if a query includes `vector` without `q`, defaulting to a pure semantic search.

### Deprecations
- The previous document template format will continue to work, but new features will only be available with the updated template that filters out non-searchable fields.
