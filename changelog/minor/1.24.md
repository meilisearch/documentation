# v1.24

First released: 2025-10-20

## New Features

### Metadata Header for Search Requests
You can now include a metadata header `Meili-Include-Metadata` in your search requests. This will add a metadata field to the response, which contains a unique identifier (`uid`) for each query, along with a reminder of the `indexUid` and its primary key.

To use this feature, include the header in your search request:

```http
GET /indexes/{indexUid}/search
Meili-Include-Metadata: true
```

The response will include the metadata:

```json
{
  "hits": [
    {
      "id": 1,
      "title": "Example Document"
    }
  ],
  "metadata": {
    "uid": "unique-query-id",
    "indexUid": "your-index-uid",
    "primaryKey": "your-primary-key"
  },
  "processingTimeMs": 2
}
```

## Improvements

### Enhanced Vector Store Search Cutoff
The interaction between the vector store and the `searchCutoffMs` setting has been improved when using the `"vectorStore": "experimental"` index setting. This enhancement allows for more efficient searches and better performance.

### Improved Compaction Behavior
The behaviors around compaction have been enhanced, leading to a more reliable interaction with task cancellation. This improvement ensures that tasks can be canceled without issues during the compaction process.

## Other
No breaking changes or deprecations were noted in this release.
