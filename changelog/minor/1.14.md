# v1.14

First released: 2025-04-14

## New Features

### Granular Filterable Attribute Settings
You now have more control over which types of filters can be disabled to optimize indexing performance. This allows you to activate only the filter features you need.

To specify which filters to enable for each attribute in your documents, use the following request:

```http
PATCH /indexes/INDEX_NAME/settings
```

With the following JSON body:

```json
{
  "filterableAttributes": [
    {
      "attributePatterns": [
        "genre",
        "artist"
      ],
      "features": {
        "facetSearch": true,
        "filter": {
          "equality": true,
          "comparison": false
        }
      }
    },
    {
      "attributePatterns": [
        "rank"
      ],
      "features": {
        "facetSearch": false,
        "filter": {
          "equality": true,
          "comparison": true
        }
      }
    }
  ]
}
```

### Composite Embedders
This feature allows you to use different embedders for search and indexing, optimizing AI-powered search performance. For example, you can use a remote embedder during indexing for higher bandwidth and a local embedder for faster search responses.

To enable this feature, follow these steps:

1. Enable the `Composite embedders` feature:

```bash
curl MEILISEARCH_URL/experimental-features \
 -H 'Content-Type: application/json' \
 -d '{"compositeEmbedders": true}'
```

2. Create an embedder with the following configuration:

```json
{
  "embedders": {
    "text": {
      "source": "composite",
      "searchEmbedder": {
        "source": "huggingFace",
        "model": "baai/bge-base-en-v1.5",
        "revision": "a5beb1e3e68b9ab74eb54cfd186867f64f240e1a"
      },
      "indexingEmbedder": {
        "source": "rest",
        "url": "https://URL.endpoints.huggingface.cloud",
        "apiKey": "hf_XXXXXXX",
        "documentTemplate": "Your {{doc.template}}",
        "request": {
          "inputs": [
            "{{text}}",
            "{{..}}"
          ]
        },
        "response": [
          "{{embedding}}",
          "{{..}}"
        ]
      }
    }
  }
}
```

3. Meilisearch will use the `indexingEmbedder` during indexing and the `searchEmbedder` for search queries.

### Retrieve Multiple Documents by ID
You can now retrieve multiple documents by their IDs with a single request:

```http
POST /indexes/INDEX_UID/documents
```

With the following JSON body:

```json
{
  "ids": [
    "cody",
    "finn",
    "brandy",
    "gambit"
  ]
}
```

The response will look like this:

```json
{
  "results": [
    {
      "id": "brandy",
      "info": 13765493
    },
    {
      "id": "finn",
      "info": 35863
    },
    {
      "id": "cody",
      "info": 122263
    },
    {
      "id": "gambit",
      "info": 22222
    }
  ],
  "offset": 0,
  "limit": 20,
  "total": 4
}
```

Note that documents are not returned in the queried order, and non-existent documents are ignored.

## Improvements

### Batch Document Requests
You can now batch together `/documents` requests using either `PUT` or `POST`, improving efficiency when handling multiple document operations.

### Timestamped Internal Indexing Steps
The `/batches` route now displays timestamped internal indexing steps, providing better visibility into the indexing process.

### Exhaustive Facet Count
An `exhaustiveFacetCount` parameter has been introduced to the `/facet-search` route, allowing you to retrieve an exhaustive count of facets.

### Reduced RAM Consumption
Improvements have been made to reduce RAM consumption during operations, enhancing overall performance.

### Experimental Feature: Cache Embeddings
An experimental feature has been added to cache embeddings during search, which can improve response times.

### Enhanced Batch Progress View
The batch progress view has been extended to include indexing of vectors, providing more detailed insights into the indexing process.

### Case Sensitivity for Armenian Characters
Armenian characters are now treated as case-insensitive, improving search accuracy for documents containing these characters.

### Avoid Reindexing on Order Changes
The system now avoids reindexing searchable attributes when their order changes, optimizing performance.

### Task Cancellation Improvements
You can now accept cancellation tasks even when the disk is full, enhancing task management capabilities.

## Other
No breaking changes or deprecations were noted in this release.
