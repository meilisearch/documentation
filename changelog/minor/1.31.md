# v1.31

First released: 2025-12-22

## Improvements

### S3-Streaming Snapshots for Enterprise Edition
The S3-streaming snapshots feature is now exclusively available for the Enterprise Edition. A license is required to use this feature in self-hosted solutions. The Community Edition will no longer support S3-streaming snapshots, but on-disk snapshots remain accessible.

### AWS IRSA Authentication for Snapshots
Support for AWS IRSA (IAM Roles for Service Accounts) has been added, allowing the use of short-lived access and secret keys for uploading snapshots to S3. This feature is available under the Enterprise Edition and can be accessed through two experimental CLI parameters.

### Strict Document Update Option
A new optional `skipCreation` boolean query parameter has been introduced for `POST` and `PUT` requests on `/indexes/{index}/documents`. When set to `true`, this parameter allows you to skip the creation of documents that do not exist in the index, preserving existing behavior when set to `false`.

```http
POST /indexes/{index}/documents?skipCreation=true
```

## Other

### Breaking Change: S3-Streaming Snapshots
The S3-streaming snapshot feature is no longer available in the Community Edition, marking a significant change in how snapshots can be managed. Users of the Community Edition who were using this feature between version 1.25 and this release will need to transition to the Enterprise Edition to continue using it.
