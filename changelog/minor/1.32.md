# v1.32

First released: 2026-01-12

## Improvements

### Skip Field-ID-Based Database Cleanup
You can now opt out of the field ID-based database cleanup when upgrading from versions prior to 1.32.0 by setting the `MEILI_EXPERIMENTAL_DISABLE_FID_BASED_DATABASES_CLEANUP` environment variable.

### Enhanced Search Performance Logging
Comprehensive progress tracking and logging for search operations have been introduced. This feature provides detailed timing information for each step of the search process, improving observability and performance analysis.

### Parallel Document Operations
Document indexing has been accelerated by processing a large number of tasks in batches or records in parallel. This change has resulted in a 7x speedup for inserting four million documents using four CPUs. Note that the `indexedDocuments` field in tasks using `skipCreation` may not accurately reflect the number of document operations performed, but the documents are still correctly indexed.

## Other

### Dumpless Upgrade Handling
The system will no longer panic when performing a dumpless upgrade on empty indexes that have configured embeddings. This change improves stability during upgrades.

### Vector Sort Improvement
The vector sort functionality has been enhanced to ensure that documents with identical similarity scores are grouped together, allowing for correct application of subsequent ranking rules.

### Document Deletion Fix
A bug has been fixed where changing `searchableAttributes` from `["*"]` to a subset of fields left orphaned data in field ID-based databases, which could cause corruption and warnings during search operations.

### Graph Link Rebuilding
The upgrade process now includes a method to rebuild graph links, addressing previously malformed graphs and ensuring proper association of related operations during upgrades.
