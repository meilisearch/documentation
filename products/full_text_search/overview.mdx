## How it works

When you add documents to Meilisearch, it:

1. **Tokenizes** text into searchable terms
2. **Indexes** terms for instant retrieval
3. **Builds** data structures for typo tolerance and prefix matching

When users search:

1. Query is processed and compared against the index
2. Results are ranked by relevancy using [ranking rules](/products/full_text_search/ranking_rules)
3. Top results are returned in milliseconds

## Quick example

```bash
# Add documents
curl -X POST "${MEILISEARCH_URL}/indexes/movies/documents" \
  -H "Authorization: Bearer ${MEILISEARCH_API_KEY}" \
  -H 'Content-Type: application/json' \
  --data-binary '[
    {"id": 1, "title": "The Dark Knight", "genre": "Action"},
    {"id": 2, "title": "Inception", "genre": "Sci-Fi"}
  ]'

# Search with typo
curl -X POST "${MEILISEARCH_URL}/indexes/movies/search" \
  -H "Authorization: Bearer ${MEILISEARCH_API_KEY}" \
  -H 'Content-Type: application/json' \
  --data-binary '{"q": "dar knigt"}'
```

Even with "dar knigt" (typos), Meilisearch returns "The Dark Knight".

## When to use full-text search

Full-text search is ideal for:

- **Product catalogs** - Search by name, description, brand
- **Content sites** - Articles, documentation, blog posts
- **User directories** - Names, emails, profiles
- **Any text-heavy dataset** - Where keyword matching matters
