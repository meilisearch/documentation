API keys control access to Meilisearch endpoints. Each key has a set of **actions** defining what operations it can perform, and an **indexes** array specifying which indexes it can access.

## Default API keys

When you set a master key, Meilisearch creates two default keys:

| Key | Actions | Use case |
|-----|---------|----------|
| **Default Search API Key** | `search` | Safe for frontend use |
| **Default Admin API Key** | `*` | Backend operations only |

<Warning>
Never expose the Admin API Key or master key in frontend code.
</Warning>

## Index restrictions

The `indexes` array limits which indexes a key can access:

```json
{
  "actions": ["search", "documents.get"],
  "indexes": ["movies", "actors"]
}
```

### Wildcard patterns

Use `*` as a wildcard at the end of index names:

```json
{
  "actions": ["search"],
  "indexes": ["movies_*"]
}
```

This grants access to `movies_en`, `movies_fr`, `movies_2024`, etc.

### All indexes

Use `["*"]` to grant access to all indexes:

```json
{
  "actions": ["search"],
  "indexes": ["*"]
}
```

## Creating a restricted key

Create a search-only key for a specific index:

```bash
curl \
  -X POST "${MEILISEARCH_URL}/keys" \
  -H 'Content-Type: application/json' \
  -H "Authorization: Bearer ${MEILISEARCH_MASTER_KEY}" \
  --data-binary '{
    "name": "Movies search key",
    "description": "Search-only access to movies index",
    "actions": ["search"],
    "indexes": ["movies"],
    "expiresAt": "2025-12-31T23:59:59Z"
  }'
```

## Best practices

### Principle of least privilege

Grant only the permissions each key needs:

```json
{
  "name": "Frontend search key",
  "actions": ["search"],
  "indexes": ["products"]
}
```

### Separate keys by function

- **Frontend**: `search` only
- **Backend indexing**: `documents.add`, `documents.delete`
- **Admin operations**: Full access via Admin key

### Use expiration dates

Set `expiresAt` for temporary access:

```json
{
  "actions": ["search"],
  "indexes": ["*"],
  "expiresAt": "2025-06-30T00:00:00Z"
}
```

### Use tenant tokens for multi-tenant apps

For user-specific search restrictions, use [tenant tokens](/products/platform/multitenancy_tenant_tokens) instead of creating individual API keys.
